# Converted App Model

An rn-mt consumer app is a normal React Native or Expo app with a generated
multi-tenant source model inside it. Treat the app as the product codebase and
rn-mt as an installed tool.

## Files you can edit

`src/rn-mt/shared` is the shared source tree. Put default behavior here when it
should apply to every label.

`src/rn-mt/tenants/<id>` contains tenant-specific full-file overrides. Put code
here when one label needs a complete replacement for a shared file.

`rn-mt.config.json` is user-owned configuration. Use it to understand labels,
targets, paths, package metadata, and command expectations.

## Files you should not edit directly

`src/rn-mt/current` is generated. It exposes the active label to the app through
facades and should be regenerated by rn-mt.

`rn-mt.generated.*` files are generated runtime or entry metadata. They should
be inspected when debugging, but durable fixes should happen in shared source,
tenant overrides, config, or rn-mt commands.

## Mental model

The app imports the current generated surface. rn-mt maps that surface back to
shared source and tenant overrides based on the active target.

The durable edit path is:

1. Change shared source when all labels should inherit the behavior.
2. Create or update a tenant override when one label needs different behavior.
3. Run `npx rn-mt sync`.
4. Validate the active target and repeat for other affected targets.

If a generated file looks wrong, inspect it to understand the symptom, then fix
the source of truth instead of patching the generated output.
