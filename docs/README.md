# rn-mt Docs

This folder is the documentation entrypoint for `rn-mt`.

If you want the fastest way to understand the package, start here:

1. `docs/package-guide.md`
2. `docs/design-decisions-handbook.md`
3. `docs/architecture.md`
4. `docs/support-policy.md`
5. `docs/roadmap.md`

What each doc is for:

- `package-guide.md`: the simple but detailed explanation of how the package works today
- `design-decisions-handbook.md`: the full decision record and product reasoning
- `architecture.md`: the package and module boundary map
- `support-policy.md`: the current support boundary for app shapes and workflows
- `roadmap.md`: the milestone and issue execution map
- `issues/0001-rn-mt-prd.md`: the product requirements document
- `issues/0002-rn-mt-brd.md`: the business requirements document

Suggested reading order:

- If you are new to the repo, read `package-guide.md` first.
- If you want the "why", read `design-decisions-handbook.md` next.
- If you want to change code, read `architecture.md` and `support-policy.md`.
- If you want to understand scope or execution history, read `roadmap.md`.
